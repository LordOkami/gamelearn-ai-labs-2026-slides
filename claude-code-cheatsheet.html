Claude Code Cheat Sheet

Basado en las mejores prácticas oficiales. Un entorno de programación agéntico que lee, planifica, ejecuta y verifica.

v3.1

Flujo de Trabajo (4 Fases)

No saltes directamente a codificar. Separa la investigación de la implementación.

1 Explorar (Plan Mode)

Pide a Claude que lea archivos y entienda el contexto sin hacer cambios.

read @src/auth and understand how we handle sessions.

2 Planificar (Plan Mode)

Pide un plan detallado. Tip: Modifícalo en tu editor antes de avanzar.

I want to add Google OAuth. What files need to change? Create a plan.

3 Implementar (Normal Mode)

Claude escribe el código basándose estrictamente en el plan.

implement the OAuth flow from your plan. Write tests and fix failures.

4 Confirmar (Normal Mode)

Usa Claude para crear commits descriptivos o PRs.

commit my changes with a descriptive message

La Regla de Oro

Verificación Autónoma

El mayor multiplicador de calidad (mejora 2x - 3x) es darle a Claude una forma de verificar su propio trabajo. Incluye comandos de prueba, linters o salidas esperadas.

Prompt Ideal:
Refactor the auth middleware. Run the test suite after making changes. Fix any failures before calling it done.

Entrevista Inversa

¿No sabes cómo estructurar tu prompt inicial? Pídele a Claude que te entreviste a ti para extraer los requisitos exactos y evitar que el agente asuma cosas incorrectas.

El Prompt Mágico: "Tengo que implementar un sistema de pagos con Stripe. Antes de escribir ningún código o plan, hazme todas las preguntas necesarias (arquitectura, casos límite, rendimiento) de una en una hasta que tengas el contexto completo."

Ventaja: Transforma una idea vaga en un documento de diseño robusto y sin alucinaciones.

Prompting Avanzado

Chain of Thought (Pensar en voz alta):

Obliga a Claude a razonar antes de codificar. Esto reduce alucinaciones en problemas complejos.

Fix the race condition in the DB wrapper. Before writing code, explain your step-by-step logic inside <thinking> tags.
Few-Shot Prompting:

Si pides un refactor, dale un ejemplo exacto de cómo era antes y cómo lo quieres ahora dentro del prompt.

.claudesignore

Archivo clave para ahorrar tokens y evitar que Claude se confunda leyendo archivos irrelevantes.

Qué bloquear siempre:
  • Dependencias (node_modules/, venv/)
  • Archivos compilados (dist/, build/)
  • Bases de datos locales (*.sqlite, *.db)
  • Secretos (.env, keys/)

Funciona exactamente igual que un .gitignore.

Enriqueciendo el Contexto

  • Archivos Directos Usa @src/utils.ts en lugar de describir rutas.
  • Imágenes y UI Pega o arrastra imágenes directamente en el prompt.
  • URLs y Permisos Pasa enlaces a docs. Usa /permissions para dominios frecuentes.
  • Canalización (Piping) Envía datos de la terminal a Claude: cat error.log | claude "fix this"

Antipatrones (Evitar)

1. La sesión "Todo Incluido"

Mezclar tareas no relacionadas ensucia el contexto.

Solución: Usa /clear entre tareas.

2. Corrección Infinita

Dejar que Claude intente arreglar el mismo error una y otra vez contamina el contexto.

Solución: Tras 2 intentos, haz /clear y da un prompt mejorado con lo aprendido.

3. Confiar y no Verificar

Aceptar código que "parece" correcto pero falla en casos límite.

Solución: Pide siempre que corra tests o scripts de validación.

El Archivo CLAUDE.md

Archivo especial en la raíz de tu proyecto que Claude lee automáticamente al iniciar.

Qué incluir:
  • Comandos de build/test
  • Convenciones de nombrado/estructura
  • Reglas de arquitectura
  • Directrices de seguridad (ej. "no alterar DB prod")
Mejor Práctica:

Mantenlo conciso. Si es muy largo, ignorará reglas importantes. Elimina instrucciones si Claude ya lo hace bien por defecto.

Skills y MCP

Amplía las capacidades de Claude integrando herramientas externas a través de Model Context Protocol (MCP).

  • Herramientas Locales: Conecta scripts de tu proyecto (ej. buscar en tu base de datos local o ejecutar scripts de limpieza).
  • APIs Externas: Dale a Claude las "skills" para interactuar con Jira, GitHub o Slack para gestionar tareas directamente.

Agents (Subagentes)

Divide problemas masivos usando múltiples agentes para no saturar el contexto principal.

Patrón de Delegación:

1. Agente Investigador: Usa un hilo solo para investigar y generar un @diseño.md.

2. Agente Ejecutor: Usa un hilo limpio con claude "implementa basándote en @diseño.md".

Hooks (Git & CI)

Claude interactúa con tu entorno real. Usa hooks para obligar a Claude a mantener la calidad del código.

Git Pre-commit Hooks: Configura Husky. Si Claude hace un commit y falla el linter, interceptará el error en la terminal y lo corregirá.

Automatización Post-run: Indícale en CLAUDE.md que siempre ejecute tus scripts de formato.

Debugging Científico

No dejes que Claude intente "adivinar" por qué falla un código basándose solo en leerlo. Oblígalo a usar instrumentación real.

El flujo ideal contra bugs:
  1. Instrumentar: "Añade console.logs detallados en la función que falla."
  2. Ejecutar: "Corre el test de integración y lee el output."
  3. Analizar: "Explica la causa raíz (root cause) del fallo basándote en los logs."
  4. Corregir: "Ahora sí, aplica la solución y limpia los logs."

Comandos y Configuración CLI

/cost
Muestra el costo de tokens consumidos en la sesión actual.
/history
Muestra el historial de prompts y acciones de la sesión.
/config
Configura el tema (light/dark) o las claves API (Anthropic).
/clear
Limpia el historial de la sesión para ahorrar contexto y evitar ruido.
/compact
Comprime la conversación para ahorrar tokens sin borrarla del todo.
/model
Cambia de modelo en vivo (ej. Sonnet para programar, Opus para pensar).
claude --dangerously-skip-permissions
Bandera avanzada: Salta la interfaz de confirmación de comandos/archivos (úsalas con alias como cc bajo tu propio riesgo para mayor velocidad).
